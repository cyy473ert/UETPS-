# 游戏功能说明文档

项目名称：Demo_cyy  
引擎版本：Unreal Engine 5.6  
文档日期：2026-06-06  
文档范围：基于当前 C++ 项目结构和已接入的蓝图资源，说明本项目已实现或已预留的核心游戏功能。

## 1. 游戏概述

本项目是一个第一人称射击生存 Demo。玩家在封闭关卡中探索、拾取武器和补给，与近战和远程敌人战斗，并通过关卡交互物推进流程。当前项目包含开始菜单、暂停菜单、HUD、背包、结算界面、存档读写、关卡解锁、角色成长、敌人 AI、武器和回血道具等系统。

当前核心体验可以概括为：

- 玩家以第一人称视角移动、瞄准、射击、换弹和切换武器。
- 玩家通过射线检测拾取武器、钥匙卡，并与电闸、安全门等关卡物件互动。
- 敌人分为近战型和远程型，具备追击、攻击、受击反馈、死亡掉落和小地图追踪能力。
- 击杀敌人获得经验，角色升级后提升生命上限倍率和伤害倍率。
- 关卡胜利后保存玩家状态和关卡进度，后续关卡可从存档恢复装备、医疗包和等级。

## 2. 玩家控制与基础状态

玩家角色由 `ACYYCharacterFather` 实现，继承自通用战斗角色 `ACombatCharacter`。玩家控制器为 `ACYYPlayerController`，使用 Enhanced Input 绑定输入行为。

### 2.1 基础操作

当前代码中支持的输入行为包括：

| 功能 | 输入行为 |
| --- | --- |
| 移动 | `MoveAction` |
| 视角旋转 | `LookAction` |
| 跳跃 | `JumpAction` |
| 暂停菜单 | `SuspendAction` |
| 背包开关 | `ToggleBackpackAction` |
| 快速使用医疗包 | `UseMedkitAction` |
| 拾取/交互 | `PickUpAction` |
| 开镜瞄准 | `AimAction` |
| 奔跑 | `RunAction` |
| 开火 | `GunShotAction` |
| 换弹 | `ReloadAction` |
| 切换 1 号武器槽 | `SwitchWeapon1Action` |
| 切换 2 号武器槽 | `SwitchWeapon2Action` |

具体键位由蓝图或 Enhanced Input 资源配置，C++ 侧负责接收动作事件并调用角色功能。

### 2.2 玩家状态

玩家状态由 `E_PlayerState` 表示，当前会根据移动、奔跑和瞄准状态自动切换：

- `Idle`：静止。
- `Walk`：普通移动。
- `Run`：向前移动并按住奔跑。
- `Aim`：开镜瞄准。

不同状态会影响移动速度：

| 状态 | 速度 |
| --- | ---: |
| 静止/行走 | 400 |
| 奔跑 | 600 |
| 瞄准 | 265 |

开镜时相机 FOV 从 90 调整为 60，停止瞄准后恢复为 90。

## 3. 武器与射击系统

武器由场景中的 `AItem` Actor 和 `UWeapon` 数据资产共同组成。`AItem` 负责拾取、挂载、枪口特效和音效接口，`UWeapon` 保存武器类型、伤害、弹药、弹匣、动画、图标和音效等数据。

### 3.1 武器数据

`UWeapon` 支持配置：

- 武器类型：例如 `RU74`、`Beretta`。
- 当前弹药和备用弹药上限。
- 基础伤害。
- 射速参数。
- 弹匣容量。
- 腰射动画、ADS 开镜射击动画、换弹动画。
- 武器图标。
- 开火、换弹、空枪音效。

### 3.2 装备槽与背包武器

玩家拥有 2 个装备槽：

- 0 号槽优先对应 `RU74`。
- 1 号槽优先对应 `Beretta`。

拾取武器时，如果对应槽位为空，武器会直接装备；如果槽位已有武器，武器会存入背包。玩家可以从背包中选择武器装备，原槽位武器会被放回背包。每个武器槽保存独立的弹匣弹药和备用弹药。

### 3.3 开火逻辑

开火流程由 `GunShot()` 和动画通知 `TriggerFireByAnimNotify()` 配合完成：

1. 玩家按下开火输入。
2. 如果当前有武器、未换弹且弹匣有子弹，则播放射击动画和音效。
3. 动画通知触发时扣除 1 发弹药。
4. 使用角色 Tick 中缓存的敌人射线结果结算命中。
5. 命中敌人后调用点伤害，并触发受击接口。
6. UI 显示命中数字。

命中判定基于相机方向射线。若命中骨骼名称包含 `head`，视为爆头，最终伤害翻倍。

最终伤害计算：

```text
最终伤害 = 武器基础伤害 * 玩家等级伤害倍率
爆头时最终伤害再 * 2
```

### 3.4 换弹逻辑

换弹由 `Reload()` 和 `FinishReload()` 实现：

- 当前武器为空时不能换弹。
- 正在换弹时不能重复换弹。
- 弹匣已满或备用弹药为 0 时不能换弹。
- 有换弹动画时，动画结束后补充弹药。
- 没有换弹动画时，使用计时器按 `ReloadDuration` 完成换弹。
- HUD 会显示换弹进度。

## 4. 生命、医疗包与成长系统

### 4.1 生命系统

生命组件为 `UHealthComponent`，支持：

- 最大生命值。
- 当前生命值复制。
- 死亡状态复制。
- 受伤事件 `OnHealthChanged`。
- 死亡事件 `OnDeath`。
- 治疗。
- 重置生命。
- 最大生命倍率。

玩家受伤时会播放受击音效，并自动关闭背包，避免战斗中 UI 阻挡操作。玩家死亡后会停止瞄准、射击和换弹，清理敌人目标 UI，并进入失败结算。

### 4.2 医疗包

背包物品由 `UInventoryComponent` 管理，目前主要管理医疗包数量。医疗包支持：

- 拾取后增加数量。
- 快捷键使用。
- 背包界面使用。
- 血量已满、无物品、当前 UI 状态不允许使用等结果反馈。

医疗包使用成功后恢复 40 点生命，并播放使用音效。

敌人死亡时有概率掉落 `AHealPickup`。回血道具被玩家碰撞拾取后，会加入玩家医疗包库存。

### 4.3 角色成长

等级组件为 `ULevelProgressionComponent`。击杀敌人后，GameMode 会读取敌人的 `XPValue` 并给玩家增加经验。

当前等级阈值：

| 等级 | 所需经验 |
| --- | ---: |
| Lv1 | 0 |
| Lv2 | 200 |
| Lv3 | 400 |
| Lv4 | 700 |
| Lv5 | 1100 |

生命倍率：

| 等级 | 生命倍率 |
| --- | ---: |
| Lv1 | 1.0 |
| Lv2 | 1.2 |
| Lv3 | 1.45 |
| Lv4 | 1.75 |
| Lv5 | 2.1 |

伤害倍率：

| 等级 | 伤害倍率 |
| --- | ---: |
| Lv1 | 1.0 |
| Lv2 | 1.1 |
| Lv3 | 1.2 |
| Lv4 | 1.35 |
| Lv5 | 1.5 |

升级时会弹出 Toast 提示，并刷新 HUD 中的等级、生命和伤害倍率信息。升级只提升生命上限倍率，不直接补满当前生命；从存档恢复时会按等级倍率重置满血。

## 5. 敌人系统

敌人基类为 `AEnemyBase`，同样继承自 `ACombatCharacter`。敌人拥有生命组件、血条、AI 控制器、怪物数据、掉落配置、音效配置和经验值。

### 5.1 敌人通用能力

敌人通用能力包括：

- 生命值和死亡判定。
- 受击音效和死亡音效。
- 受击后把攻击者设置为目标。
- 头顶血条显示。
- 小地图注册与注销。
- 死亡后按概率掉落医疗包。
- 死亡后向玩家提供经验值。

头顶血条在玩家距离敌人不超过 `OverheadHealthBarDistance` 时显示，默认距离为 1500。

### 5.2 怪物数据资产

`UMonster` 用于配置敌人参数：

- 类型：近战或远程。
- 最大生命值。
- 移动速度。
- 发现玩家范围。
- 丢失目标范围。
- 攻击范围。
- 攻击间隔。
- 单次攻击伤害。
- 近战命中半径。
- 远程攻击前摇。
- 远程射线距离。
- 远程散布。
- 单次攻击发射数量。
- 远程期望距离。
- 远程受击后撤距离。

敌人实例还支持在关卡中单独覆盖生命、速度、伤害和攻击间隔，方便做不同强度的敌人。

### 5.3 近战敌人

近战敌人由 `AMeleeEnemyCharacter` 实现，具备：

- 巡逻点配置。
- 侦测范围。
- 近战攻击范围。
- 左右手碰撞盒。
- 攻击蒙太奇。
- 受击反应蒙太奇。
- 攻击冷却。
- 单次攻击只结算一次伤害。

近战攻击通过手部碰撞盒与玩家重叠来结算，攻击窗口由动画通知或蓝图事件控制开启和关闭。

### 5.4 远程敌人

远程敌人由 `ARangedEnemyCharacter` 实现，具备：

- 侦测范围。
- 战斗范围。
- 行为树。
- 瞄准前摇。
- 射线攻击。
- 命中散布。
- 攻击冷却。
- 受击反应。
- 受击或距离过近时后撤。
- 战斗横移。

远程敌人会尝试与玩家保持合适距离。过近时可后撤，战斗中可横移，使战斗节奏区别于近战敌人。

## 6. 交互与关卡推进

交互统一通过 `IInteract` 接口处理。玩家每帧从相机向前做短距离交互射线，检测实现了交互接口的 Actor，并显示或隐藏交互提示。

交互射线距离默认为 250。

### 6.1 可拾取武器

武器 `AItem` 实现交互接口：

- 被玩家瞄准时显示拾取 UI。
- 交互后挂载到玩家手臂武器插槽。
- 关闭碰撞，避免装备后继续被拾取射线命中。
- 根据槽位规则进入装备槽或背包。
- 播放拾取和装备音效。
- 显示拾取 Toast。

### 6.2 电闸

`APowerSwitch` 表示电力开关：

- 玩家交互后只触发一次。
- 设置 `UShelterGameInstance::bPowerRestored = true`。
- 隐藏交互提示。
- 播放启动音效。

### 6.3 钥匙卡

`AKeyCardItem` 表示钥匙卡：

- 玩家交互后设置 `UShelterGameInstance::bHasKeyCard = true`。
- 显示获得提示。
- 播放拾取音效。
- 销毁自身。

### 6.4 安全门

`ALockedDoor` 表示锁门：

- 初始状态为未解锁，状态灯为红色。
- 当 GameInstance 中同时满足 `bPowerRestored` 和 `bHasKeyCard` 时解锁。
- 解锁后状态灯变为绿色。
- 玩家交互后门体按配置角度旋转打开。
- 支持开门音效。

### 6.5 终点触发器

`ALevelEndTrigger` 表示关卡终点区域：

- 玩家进入触发盒后调用 GameMode 的 `TriggerVictory()`。
- 触发一次后不重复触发。

## 7. 关卡流程与胜负条件

关卡逻辑由 `ACYYGameModeBase` 管理。GameMode 会在开始时根据当前地图名推断关卡编号：

- 地图名包含 `Level1` 时，关卡编号为 1。
- 地图名包含 `Level2` 时，关卡编号为 2。

### 7.1 失败条件

玩家生命归零时触发失败结算，失败原因记录为 `PlayerDead`。

### 7.2 胜利条件

当前胜利方式包括：

- 进入 `ALevelEndTrigger` 终点区域后胜利。
- 在第 2 关中，所有敌人被清空后自动胜利。

胜利后 GameMode 会保存进度，并显示结算界面。

### 7.3 结算数据

结算数据包含：

- 胜利或失败。
- 击杀数量。
- 本局耗时。
- 结束原因。
- 时间戳。

结算界面支持重试、返回菜单和进入下一关等流程动作。

## 8. UI 系统

UI 由 `UUIManagerComponent` 统一管理，挂载在 `ACYYPlayerController` 上。它负责 UI 创建、状态切换、输入模式切换、HUD 刷新和流程按钮响应。

### 8.1 UI 状态

项目中存在以下主要 UI 状态：

- 开始菜单。
- 游戏中 HUD。
- 暂停菜单。
- 背包界面。
- 结算界面。

UI 状态切换会处理鼠标显示、输入模式、暂停和 Widget 显隐。

### 8.2 HUD 内容

HUD 支持：

- 玩家生命显示。
- 弹匣弹药和备用弹药显示。
- 换弹进度显示。
- 武器槽显示和当前槽高亮。
- 敌人目标血条。
- 命中数字。
- 爆头命中数字区分。
- 等级、当前生命和伤害倍率信息。
- 小地图。
- Toast 提示。

玩家瞄准敌人时，敌人血条会在 HUD 中显示；移开后清除。

### 8.3 背包界面

背包界面会汇总：

- 医疗包。
- 已装备武器。
- 存放在背包中的武器。

背包条目包含图标、名称、描述、数量、是否可使用、禁用原因、武器类型、弹药信息、装备状态和背包索引等数据。玩家可在背包中使用医疗包，或将背包武器装备到对应槽位。

战斗操作如受伤、开火、瞄准、拾取等会自动关闭背包。

## 9. 小地图系统

敌人在 `BeginPlay` 时注册到 `UMinimapTrackingSubsystem`，在 `EndPlay` 时注销。UI 侧通过 `UMinimapWidget` 显示小地图信息。该设计将敌人追踪从敌人本体和 HUD 中拆分出来，便于后续添加玩家、任务点、出口等标记。

## 10. 存档与关卡解锁

存档对象为 `UCYYSaveGame`，存档管理器为 `FCYYSaveManager`。

### 10.1 存档内容

当前存档保存：

- 已解锁最高关卡。
- 当前关卡。
- 第 1 关是否完成。
- 第 2 关是否完成。
- 医疗包数量。
- 玩家等级。
- 玩家经验。
- 已装备武器类型列表。
- 背包武器类型列表。
- 存档时间戳。

### 10.2 保存时机

胜利时 GameMode 会自动：

1. 加载或创建存档。
2. 从玩家身上收集医疗包、武器、等级和经验。
3. 标记当前关卡完成。
4. 设置继续游戏时进入下一关。
5. 写入磁盘。

### 10.3 读取与恢复

进入第 2 关及之后时，玩家 `BeginPlay` 会尝试读取存档并恢复：

- 医疗包数量。
- 装备槽武器。
- 背包武器。
- 玩家等级。
- 玩家经验。
- 根据等级倍率恢复生命上限并重置满血。

恢复武器时会根据保存的武器类型，从角色上的 `WeaponClassMap` 生成对应武器 Actor，再放回装备槽或背包。

## 11. 数据驱动与扩展点

本项目已经将关键内容拆成可配置数据或蓝图资源，便于扩展：

### 11.1 新增武器

新增武器通常需要：

1. 创建或配置 `UWeapon` 数据资产。
2. 设置武器类型、伤害、弹药、弹匣、动画、图标和音效。
3. 创建继承 `AItem` 的武器蓝图。
4. 绑定武器网格、拾取提示、枪口特效、拾取和装备音效。
5. 在玩家角色的 `WeaponClassMap` 中配置武器类型到蓝图类的映射。
6. 如需专属槽位规则，需要扩展 `ResolveWeaponEquipSlot()`。

### 11.2 新增敌人

新增敌人通常需要：

1. 创建 `UMonster` 数据资产。
2. 选择近战或远程类型。
3. 配置生命、速度、侦测、攻击和伤害参数。
4. 创建近战或远程敌人蓝图。
5. 配置行为树、攻击动画、受击动画、血条 Widget、音效和掉落。
6. 如需特殊强度，可在关卡实例中开启 Override 参数。

### 11.3 新增关卡

新增关卡通常需要：

1. 配置对应 GameMode。
2. 放置玩家出生点、敌人、武器、医疗包、交互物和终点触发器。
3. 如使用自动关卡编号，需要保证地图名包含可识别的 Level 编号，或扩展 GameMode 中的关卡推断逻辑。
4. 更新 UIManager 的关卡名映射和存档解锁逻辑。

### 11.4 新增交互物

新增交互物应实现 `IInteract`：

- `ShowInteractWidget()`：显示交互提示。
- `HideInteractWidget()`：隐藏交互提示。
- `OnInteract()`：执行交互逻辑。
- `DataAsset()`：武器类使用，普通交互物可返回空。

玩家无需额外知道交互物类型，只要射线命中实现接口的 Actor，就能显示提示并交互。

## 13. 注意事项

- 当前代码中部分中文日志或文本存在编码异常，建议统一源文件编码为 UTF-8，避免编辑器中出现乱码。
- 输入动作的具体键位不在 C++ 中硬编码，需要在 Enhanced Input 资源中确认。
- 存档当前保存武器类型，不保存每把武器的精确弹药数量；恢复时会按数据资产弹匣和最大弹药重新初始化。
- GameMode 当前通过地图名包含 `Level1` 或 `Level2` 推断关卡编号，后续增加更多关卡时建议改为显式配置。
- 敌人血条和小地图依赖 Widget、Subsystem 与蓝图配置，关卡中新建敌人时需要确认相关类已正确设置。
